JSON

Es un formato de texto que forma parte del sistema de JavaScript y que se deriva de su sintaxis, pero no tiene como objetivo la creación de programas, sino el acceso, almacenamiento e intercambio de datos. La sintaxis JSON se deriva de la sintaxis de notación de objetos de JavaScript, pero el formato JSON es solo texto. El código para leer y generar datos JSON se puede escribir en cualquier lenguaje de programación.

Ejemplo JSON

{
  "employees":[
    {"firstName":"John", "lastName":"Doe"},
    {"firstName":"Anna", "lastName":"Smith"},
    {"firstName":"Peter", "lastName":"Jones"}
  ]
}

El formato JSON es sintácticamente idéntico al código para crear objetos JavaScript. Debido a esta similitud, un programa JavaScript puede convertir fácilmente datos JSON en objetos JavaScript nativos.

Reglas de sintaxis JSON

  • Los datos están en pares nombre/valor.
  • Los datos están separados por comas.
  • Las llaves sostienen objetos.
  • Los corchetes contienen matrices.

Datos JSON: un nombre y un valor

Los datos JSON se escriben como pares nombre/valor, al igual que las propiedades de los objetos de JavaScript. Un par nombre/valor consta de un nombre de campo (entre comillas dobles), seguido de dos puntos, seguido de un valor: 

"firstName":"John"

Los nombres JSON requieren comillas dobles. Los nombres JavaScript no.

Objetos JSON

Un objeto contiene una clave y un valor. Hay dos puntos después de cada clave y una coma después de cada valor, que también distingue a cada objeto. Ambos están entre comillas. 

{
  "empleados":{
    "nombre":"Tom",
    "apellido":"Jackson"
  }
}

Matrices JSON

Las matrices JSON se escriben dentro de corchetes.

"employees":[
  {"firstName":"John", "lastName":"Doe"},
  {"firstName":"Anna", "lastName":"Smith"},
  {"firstName":"Peter", "lastName":"Jones"}
]

Número

El número en JSON debe ser un número entero o un punto flotante, como:

{"Edad":"30"}

Booleano

Puedes usar verdadero o falso como valor, de la siguiente manera:

{"Casado":"false"}

Nulo

Es para mostrar que no hay información:

{"Tipo de sangre":"null"}

Convertir un texto JSON en un objeto JavaScript

Un uso común de JSON es leer datos de un servidor web y mostrarlos en una página web. Para simplificar, esto se puede demostrar utilizando una cadena como entrada.

Primero, crea una cadena de JavaScript que contenga la sintaxis JSON:

let text = '{ "employees" : [' +
'{ "firstName":"John" , "lastName":"Doe" },' +
'{ "firstName":"Anna" , "lastName":"Smith" },' +
'{ "firstName":"Peter" , "lastName":"Jones" } ]}';

Luego, usa la función incorporada de JavaScript JSON.parse() para convertir la cadena en un objeto JavaScript: 

const obj = JSON.parse(text);

Por último, utiliza el nuevo objeto JavaScript en tu página:

<p id="demo"></p>

<script>
document.getElementById("demo").innerHTML =
obj.employees[1].firstName + " " + obj.employees[1].lastName;
</script>

JSON es un formato sencillo y liviano

Se caracteriza por:

  • Ser fácil de aprender y solucionar problemas.
  • Tener una sintaxis similar a JavaScript.
  • Ser independiente del lenguaje, por lo que se puede usar en una gran variedad de lenguajes de programación.
  • Estar escrito en archivos de texto plano, lo que facilita su transporte en redes como Internet.

Cinco principales casos de uso de JSON

  1. Transferencia de datos entre sistemas: JSON es ideal para transferir datos entre diferentes sistemas y lenguajes de programación. Por ejemplo, una compañía puede enviar datos de dirección en formato JSON a una API de validación de direcciones.
  2. Generación de un objeto JSON a partir de datos generados por el usuario: JSON es ideal para el almacenamiento de datos temporales, como formularios enviados en sitios web.
  3. Configuración de datos para aplicaciones: Las credenciales y configuraciones de aplicaciones se pueden almacenar en archivos JSON.
  4. Simplificación de modelos de datos complejos: JSON facilita la extracción de datos de documentos complejos.
  5. Archivos de configuración y almacenamiento de datos: JSON admite estructuras anidadas y matrices, lo que lo hace ideal para almacenar datos complejos y jerárquicos.
